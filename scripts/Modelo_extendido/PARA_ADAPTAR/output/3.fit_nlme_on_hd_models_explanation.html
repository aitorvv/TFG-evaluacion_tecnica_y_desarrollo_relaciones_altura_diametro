Overview

This script is designed to fit Non-Linear Mixed-Effects (NLME) models on height-diameter (HD) data for trees. It iterates through tree species and predefined top models to evaluate combinations of fixed and random effects, storing the resulting model objects on disk and gathering fitting statistics.

1. Data and Functions Loading

  • Loads required packages (nlme, tidyverse, broom.mixed) and internal functions for HD equations (0.support_functions.r and 0.hd_equations_collection.r).
  • Prepares the main dataset (swodata.csv) by normalizing variable names (e.g., ht mapped to h).
  • Merges the top selected HD models with their previously estimated baseline coefficients (1.fit_base_hd_models_coefs.csv).

2. Configuration of Effects

  • Defines alternative fixed effects to evaluate: null (1), crown competition factor (ccfl), basal area in larger trees (bal), and both (ccfl + bal).
  • Defines alternative random effects: none (0) or stand-level (standid).

3. Nested Fitting Loops

The execution relies on a multi-level iterative process: - Species level: Filters data for a specific species. - Model level: Iterates over the selected top HD models for that species, loading their specific functions and base coefficients. - Fixed effects combinations: Dynamically creates a grid of all parameter permutations against the defined fixed effects using expand.grid(). - Random effects combinations: Similarly computes combinations to selectively apply the stand-level random effect to different parameters.

4. Model Fit and Persistence

  • Constructs the starting values dynamically using model.matrix() to handle categorical/continuous covariates properly.
  • Fits either an nlme (mixed effects) or a gnls (generalized nonlinear least squares, when no random effects are included) model.
  • Calculates and stores performance statistics (R-squared, RMSE, MAE, bias, AIC, BIC) using predict() and custom functions.
  • Saves each successfully fitted model as an .rds file to prevent RAM overload.

Technical Considerations

Dynamic Formula Generation

The script dynamically pieces together formula strings (e.g., h ~ model_function(a, b, dbh)) and lists of formulas for fixed arguments based on the target model's parameter names. This enables the script to adapt gracefully to HD models that have any number of coefficients (e.g., 2-parameter vs. 3-parameter equations).

    # get model coefficients
    start_values <- species_top_models %>% dplyr::filter(model == model_name)
    start_values_list <- as.list(setNames(start_values$estimate, start_values$term))
    model_coefs <- start_values_list[order(names(start_values_list))]

    # get model details
    model_function <- hd_models_collection[[model_name]]$func
    coefs_labels <- names(model_coefs)
    formula_str <- paste0("h ~ model_function(", paste(coefs_labels, collapse = ", "), ", dbh)")
    model_formula <- as.formula(formula_str)

Memory Management

Storing thousands of fitted nlme models in a single R list or environment would quickly consume all available RAM. The script uses saveRDS() to write each successfully fitted model directly to disk inside output/3.nlme_hd_models/, keeping only the lightweight metadata and statistics in the results_stats dataframe.

          # save model to disk to prevent out of memory issues
          saveRDS(fit, file = paste0("output/3.nlme_hd_models/", sp, "_", model_name, "_fe_", i, "_re_", j, ".rds"))

Error Handling

Fitting nonlinear models frequently fails to converge depending on the initial values and data structure. The code wraps the fitting functions within a tryCatch() block. If convergence fails, it logs a warning, skips saving the .rds file, and records NA stats with an error = TRUE flag in the results table, allowing the loop to continue without crashing the execution.

        cat("  - fitting fe combination ", i, " and re combination ", j, "...\n", sep = "")
        fit_result <- tryCatch(
          {
            # if the random effect combination is "none", fit gnls instead of nlme
            if (inherits(re_combi, "character") && re_combi == "none") {
              fit <- nlme::gnls(
                model = model_formula,
                data = species_data,
                params = fe_combi,
                start = start_vec
              )
            } else {
              fit <- nlme::nlme(
                model = model_formula,
                data = species_data,
                fixed = fe_combi,
                random = re_combi,
                start = start_vec
              )
            }
          },
          error = function(e) {
            cat("WARNING: Fitting failed for model ", model_name, " fe: ", i, " re: ", j, " - ", e$message, "\n", 
                sep = "")
            return(NULL) # return NULL if there's an error
          }
        )

Model Type Fallback

Because some combinations do not include random effects (identified by all parameters having 0), the script seamlessly switches from using nlme::nlme() to nlme::gnls() (Generalized Nonlinear Least Squares). This ensures that base nonlinear fits are comparable to the mixed-effects versions within the same pipeline.

            # if the random effect combination is "none", fit gnls instead of nlme
            if (inherits(re_combi, "character") && re_combi == "none") {
              fit <- nlme::gnls(
                model = model_formula,
                data = species_data,
                params = fe_combi,
                start = start_vec
              )
            } else {
              fit <- nlme::nlme(
                model = model_formula,
                data = species_data,
                fixed = fe_combi,
                random = re_combi,
                start = start_vec
              )
            }

Model Matrix Starting Values

Adding covariates (factors or continuous like crown density ccfl) adds parameters. To ensure convergence, the solver needs accurate initial guesses for these new slopes. The script builds a model.frame and its corresponding model.matrix, defaulting starting estimates for new parameters to 0 while holding core non-linear coefficients to baseline priors.

      # build the start vector dynamically
      start_vec <- c()
      for (k in seq_along(fe_combi)) {
        # extract parameter name from the left-hand side of the formula (e.g., a0 from a0 ~ 1 + ccfl)
        p_name <- as.character(fe_combi[[k]][[2]])

        # the base starting value from the non-linear squares coefficient list
        base_val <- model_coefs[[p_name]]

        # extract the rhs of the formula (e.g., ~ 1 + ccfl)
        rhs_formula <- formula(delete.response(terms(fe_combi[[k]])))

        # dynamically build model matrix to support continuous or categorical covariates (factors)
        # model.matrix handles factors automatically, outputting k-1 columns for k levels
        m_frame <- model.frame(rhs_formula, data = species_data, na.action = na.pass)
        m_matrix <- model.matrix(rhs_formula, data = m_frame)
        n_extra_params <- ncol(m_matrix) - 1

        # append the base value, and fill with 0s for the extra covariates
        start_vec <- c(start_vec, base_val, rep(0, n_extra_params))
      }
